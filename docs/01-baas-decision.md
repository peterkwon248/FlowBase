## 1. 후보

| | Supabase | bkend.ai | Firebase |
|---|---|---|---|
| DB | Postgres (관계형) | Postgres (관계형) | Firestore (document) |
| 인증 | 내장 (이메일/OAuth/Magic Link) | 내장 (한국어 친화) | 내장 (Google 강함) |
| 실시간 | Realtime (Postgres replication) | 지원 | Firestore listener |
| Row-level Security | ✅ 강력 | ✅ | 별도 룰 언어 |
| 한국어 문서 | 부분 | ✅ 풍부 | 부분 |
| 한국 결제 | 직접 통합 필요 | 토스페이 가이드 있음 | 직접 통합 필요 |
| 셀프호스팅 | ✅ (Docker) | ⚠️ 확인 필요 | ❌ |
| 시장 친숙도 | ✅ 높음 | 신생 | ✅ 매우 높음 |
| 가격 (개발자 1) | $0 (Free) → $25/mo (Pro) | 확인 필요 | $0 → 사용량 |
| 본 프로젝트와 호환 | Next.js 공식 가이드 多 | bkit 생태계 ↔ 본 워크플로우 일치 | 가능하나 NoSQL 고려 |
| 본 도구의 데이터 모델 (관계 많음) | ✅ 적합 | ✅ 적합 | ⚠️ 관계 비싸짐 |

**Firebase 탈락**: 이 도구의 핵심이 *관계형 스키마* 인데 NoSQL은 자살. 빠짐.

→ **결승: Supabase vs bkend.ai**

| 기준 | 가중치 | 사유 |
|---|---|---|

## 3. 시나리오별 점수 (예측)

### Supabase 선택 시
- 장: 자료 풍부, Stack Overflow 커버리지 큼, Postgres 표준 그대로
- 단: 한국어 문서 부족, 토스페이 직접 통합 필요, 한국 호스팅 리전 가까움 ([ap-northeast](http://ap-northeast)-2 = 서울 가능)

### bkend.ai 선택 시
- 장: bkit 생태계와 정렬, 한국어 문서, 토스페이 가이드, **이 도구의 워크플로우 동일 사용자층**
- 단: 신생 → 트러블슈팅 시 자료 적음, 락인 위험 약간 더 높음

## 4. 권장

> **bkend.ai를 우선 검토, 단 다음 3개 확인 후 확정.**

확인할 것:
1. **DB 스키마 마이그레이션 도구** — Drizzle / Prisma / 자체 마이그레이션 중 무엇을 지원?
2. **RLS** — 워크스페이스/멤버 단위 권한이 RLS로 깔끔히 표현 가능?
3. **무료 한도와 가격 곡선** — Free에서 100명, Pro에서 1000명 시 비용 예측

이 셋이 ❌면 → **Supabase 폴백**.

**왜 bkend.ai를 우선시?**
- 본 도구의 차별점("AI 초안 + 한국어 1차")과 BaaS 시장 포지션이 가장 잘 맞음
- bkit 워크플로우와 자연스럽게 연결됨
- 한국어 사용자가 가입 → 결제까지 흐름이 매끄러움
